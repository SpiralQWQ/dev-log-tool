# 程序员日志模板 — 管线阶段

> **所属管线**：Phase 2（执行）
> **入口条件**：数据采集（[COLLECT]）和 JSON 存档（[ARCHIVE]）已完成
> **前置读取**：`data_toolchain.md`（采集工具链）

---

## 步骤 1：检查同天日志

- **动作**：扫描 `程序员日志/` 下是否有今天日志（文件名含 `YYYYMMDD`）
- **判断**：有→合并；无→新建
- **状态输出**：`[LOG] 同天日志: 存在/不存在`

## 步骤 2：写入 8 字段头部

**顺序固定，缺一不可。未知填"未记录"，禁止留空：**

```
日期: YYYY-MM-DD（时段）
会话起止: 约 HH:MM → HH:MM UTC+8
模型: [模型名]
Token 消耗: [汇总本次会话模型调用，格式见 billing_rules.md]
花费反馈: [汇总本次会话模型调用，格式见 billing_rules.md]
变更记录: [摘要] | 编码 Xmin / 浏览 Xmin / 通信 Xmin / 其他 Xmin / 终端 Xmin / 文档 Xmin / 设计 Xmin | +N/-N/~N
文件状态: +N / ~N / -N
量化快照: {语言} {行}行 · {文件}文件 · {作者}作者
```

- **状态输出**：`[LOG] 8字段头部: 完成`

## 步骤 3：时间分类展开

- **规则**：全部列出，禁止合并到"其他"
- **格式**：`编码 Xmin / 浏览 Xmin / 通信 Xmin / 其他 Xmin / 终端 Xmin / 文档 Xmin / 设计 Xmin`
- **状态输出**：`[LOG] 时间分类: 已展开`

## 步骤 4：密钥遮蔽

- **规则**：API Key/Token/密码 必须 `***` 打码
- **检查范围**：日志正文 + JSON 存档
- **状态输出**：`[MASK] 密钥遮蔽: 通过`

## 出口条件

- 输出物：`NN_关于<概要>的日志_<工作区>_YYYYMMDD_HHmm.md`
- 签章：`[LOG] 日志完成`

## 质检门禁（Phase 3 — Qwen + GLM 各自独立审查，均≥9.5 方通过）

### 检查项细则

| 检查项 | 具体判定标准 | Qwen 评分 | GLM 评分 |
|--------|------------|:---------:|:--------:|
| 8 字段完整 | 逐字段核对：日期/会话起止/模型/Token/花费/变更记录/文件状态/量化快照。缺失/留空/填"未记录"但实际可获取 → 扣分 | /10 | /10 |
| 时间分类展开 | 编码/浏览/通信/其他/终端/文档/设计 7 类全部列出。合并到"其他" → 扣 2 分 | /10 | /10 |
| 密钥无泄露 | 全文搜索 `sk-` `$env:*KEY*` `password=` `Bearer ` `eyJ` 模式。发现明文 → 扣 5 分 | /10 | /10 |
| 同天合并 | 当天有旧日志时是否追加而非覆盖。覆盖 → 扣 3 分 | /10 | /10 |

### 通过条件

```
Qwen 评分 ≥ 9.5 AND GLM 评分 ≥ 9.5 → [质检] 裁决: 通过
Qwen 评分 < 9.5 OR GLM 评分 < 9.5 → [质检] 裁决: 返工（标注扣分项）
```
